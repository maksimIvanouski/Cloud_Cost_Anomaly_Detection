# 📂 Data Directory

This directory contains all datasets used by the Cloud Cost Anomaly Detection project.

## Directory Structure

```
data/
├── raw/
│   └── cloud_cost_data.csv       # Generated by src/data_generation.py
├── processed/
│   └── cloud_cost_features.csv   # Generated by src/feature_engineering.py
└── README.md
```

## Raw Data — `raw/cloud_cost_data.csv`

The raw dataset is generated synthetically by `src/data_generation.py` using `RANDOM_STATE=67` to ensure full reproducibility. It simulates approximately 90 days of daily cloud infrastructure billing records across multiple providers, regions, services, and environments.

- **Records:** ~5,000
- **Anomaly rate:** ~7% (reflecting natural class imbalance in anomaly detection tasks)

### Column Descriptions

| # | Column | Type | Description |
|---|--------|------|-------------|
| 1 | `date` | date | Record date |
| 2 | `day_of_week` | int | Day of the week (0=Monday, 6=Sunday) |
| 3 | `is_weekend` | int | Binary flag: 1 if Saturday or Sunday, 0 otherwise |
| 4 | `month` | int | Month of the year (1–12) |
| 5 | `cloud_provider` | str | Cloud provider: AWS, Azure, or Google Cloud |
| 6 | `region` | str | Cloud region (e.g., us-east-1, eu-central-1) |
| 7 | `service_name` | str | Cloud service (e.g., EC2, S3, RDS, Lambda, CloudWatch, Data Transfer, Kubernetes) |
| 8 | `environment` | str | Deployment environment: production, staging, or development |
| 9 | `usage_amount` | float | Abstract usage units |
| 10 | `resource_count` | int | Number of active resources |
| 11 | `data_transfer_gb` | float | Data transferred in gigabytes |
| 12 | `storage_gb` | float | Storage consumed in gigabytes |
| 13 | `compute_hours` | float | Compute time in hours |
| 14 | `request_count` | int | API/request count |
| 15 | `previous_day_cost` | float | Cost on the previous day (USD) |
| 16 | `rolling_3_day_avg_cost` | float | Mean cost over the past 3 days (USD) |
| 17 | `rolling_7_day_avg_cost` | float | Mean cost over the past 7 days (USD) |
| 18 | `cost_usd` | float | Total cost in USD (current day) |
| 19 | `anomaly_type` | str | Anomaly scenario label (for analysis only — **NOT** used as a training feature) |
| 20 | `is_anomaly` | int | Target variable: 0 = normal, 1 = anomaly |

## Processed Data — `processed/cloud_cost_features.csv`

The processed dataset is produced by `src/feature_engineering.py` after applying feature engineering to the raw data. It includes all original columns plus derived features such as cost trend ratios, efficiency metrics, and rule-based indicators.

## Anomaly Types

The following 8 anomaly scenarios are injected into the synthetic dataset:

| Anomaly Type | Description |
|---|---|
| `sudden_compute_spike` | Compute cost jumps 4–8x above normal levels |
| `abnormal_data_transfer` | Data transfer increases 10–20x above baseline |
| `storage_growth` | Storage consumption grows 8–15x abnormally |
| `forgotten_test_environment` | Development/staging costs escalate to production levels |
| `unexpected_region_usage` | Non-primary region shows unusually high cost |
| `monitoring_cost_spike` | Monitoring/logging costs spike 5–10x |
| `weekend_production_activity` | Unexpected high production costs during weekends |
| `duplicated_resource_deployment` | Resources duplicated 4–8x beyond normal count |

## Reproducibility

All data is generated with `RANDOM_STATE=67`. Running `src/data_generation.py` on the same machine will produce identical output, enabling fully reproducible experiments.

## Notes

- The `anomaly_type` column is included for analysis and interpretability purposes only. It is **never** used as an input feature during model training.
- Rolling average features (`rolling_3_day_avg_cost`, `rolling_7_day_avg_cost`) are computed using only past values to prevent data leakage.
- The `.gitignore` excludes CSV files from version control. Regenerate them by running the pipeline scripts.
